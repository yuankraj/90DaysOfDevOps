# Day 03 – Linux Commands Practice

## Overview
Today's goal was to build confidence using essential Linux commands that are frequently used in real-world DevOps environments.

Linux command-line skills are critical for troubleshooting production systems, analyzing logs, monitoring processes, and diagnosing networking issues.

This cheat sheet contains commonly used commands grouped into three main categories:
- Process Management
- File System Operations
- Networking Troubleshooting

## What I Learned
Through this exercise, I practiced commands that help with:
- Monitoring running processes
- Managing files and directories
- Troubleshooting network connectivity
- Inspecting system resources

Understanding these commands improves my ability to quickly debug issues on Linux servers.

## Commands Covered
The cheat sheet includes more than **20 commonly used commands**, such as:

### Process Management
- ps
- top
- htop
- kill
- pkill
- systemctl

### File System
- ls
- cd
- pwd
- mkdir
- rm
- cp
- mv
- cat
- less
- chmod
- df
- du

### Networking
- ping
- ip addr
- curl
- dig
- netstat
- ss

## Why This Matters for DevOps
DevOps engineers spend a large portion of their time working directly in Linux environments.

Mastering these commands helps with:
- Faster troubleshooting
- Debugging crashed services
- Identifying resource issues
- Diagnosing network problems

These skills are fundamental for managing production infrastructure.
