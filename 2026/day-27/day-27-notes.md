# Task 1 – Profile Audit (Before Changes)

Before improving my GitHub profile, I reviewed it from the perspective of a recruiter or hiring manager.

### Observations

- Profile picture: Basic profile picture present but could be more professional.
- Bio: Added a short description about learning DevOps.
- Pinned repositories: Some were random projects and forks.
- Repository descriptions: Some repositories did not have clear descriptions.
- Documentation: A few repositories were missing proper README files.

### First Impression

A recruiter might not fully understand my learning journey or the projects I am working on.  
The profile needed better organization and clearer documentation.

---

# Task 2 – Profile README

I created a special repository with the same name as my GitHub username.

Example:

```
github.com/<your-username>/<your-username>
```

This repository contains a **README.md** that appears on my GitHub profile.

### Profile README Content

My README includes:

- Introduction about me
- Current learning journey (90 Days of DevOps)
- Skills I am learning
- Important repositories
- Contact information
