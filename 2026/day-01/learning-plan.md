90 Days DevOps Learning Plan
Current Level

I am a B.Tech 6th semester student with basic knowledge of Linux, Python, and DevOps. I also have some exposure to cybersecurity and CTF challenges, which helped me understand Linux systems and troubleshooting.

My Understanding of DevOps & Cloud Engineering

DevOps is a practice that combines software development and IT operations using automation, CI/CD pipelines, containers, and monitoring to deliver software faster and more reliably.

Cloud Engineering focuses on designing, deploying, and managing scalable infrastructure on cloud platforms like AWS, Azure, or GCP.

Why I Am Starting DevOps & Cloud

I want to build strong real-world infrastructure and deployment skills beyond just writing code.

Learning DevOps and Cloud will help me:

Understand how production systems run

Automate software delivery

Prepare for DevOps / Cloud Engineer roles in the future

My 90-Day Goals

Deploy a containerized web application using Docker and Kubernetes.

Build a CI/CD pipeline using GitHub Actions or Jenkins.

Deploy an application on a cloud platform (AWS or GCP).

Core DevOps Skills I Want to Build

Linux system troubleshooting and command-line mastery

CI/CD pipeline creation and automation

Containerization and Kubernetes deployment

Weekly Time Commitment

To stay consistent, I will dedicate:

Weekdays: 2 – 2.5 hours per day

Weekends: 4 – 6 hours per day

This will give me approximately 18–20 hours per week for learning and practice.

How I Will Stay Consistent

Follow a daily learning schedule and complete small tasks every day.

Practice through hands-on labs and projects, not just theory.

Final Goal

By the end of these 90 days, I aim to build real DevOps projects, understand cloud infrastructure, and be confident deploying production-ready applications using modern DevOps tools.
